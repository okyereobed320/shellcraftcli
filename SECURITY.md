# 🔐 Security Policy

## 📢 Reporting a Vulnerability

The Shellcraft project takes security seriously.
If you discover a security vulnerability, please report it **responsibly and privately**.

### 🚨 Do NOT:

* Open a public GitHub issue for security vulnerabilities
* Disclose the issue publicly before it has been reviewed

---

### 📬 How to Report

Please send details to:

**Email:** [security@shellcraft.dev](mailto:security@shellcraft.dev) *(replace with your actual email)*

Include:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Suggested fix (if available)

---

## ⏱️ Response Timeline

We aim to:

* **Acknowledge** your report within **48 hours**
* **Investigate** and validate within **3–5 days**
* **Release a fix** as soon as possible depending on severity

---

## 🛡️ Supported Versions

Currently, Shellcraft is under active development.

| Version              | Supported |
| -------------------- | --------- |
| Latest (main branch) | ✅         |
| Older versions       | ❌         |

---

## 🔍 Scope

This policy applies to:

* CLI commands
* Simulation environments
* Data files and execution logic
* Dependency vulnerabilities

---

## 🙏 Responsible Disclosure

We appreciate responsible disclosure and will:

* Credit contributors (if desired)
* Work quickly to resolve issues
* Keep communication transparent

---

## ⚖️ Legal

Please:

* Act in good faith
* Avoid data destruction, service disruption, or privacy violations
* Only test against your own environments

---

## 💡 Security Philosophy

Shellcraft is built to be:

* Safe for learning
* Secure by design
* Reliable for real-world simulation
